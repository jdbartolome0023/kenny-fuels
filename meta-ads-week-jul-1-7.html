🔒
Permanent, not a bug
  • Graphedia confirmed visitors who decline marketing cookies cannot legally be tracked for ad attribution, by any method, under Irish ePrivacy/GDPR rules.
  • This is a permanent limitation, not a bug in progress.
  • A server-side Conversions API build is underway, but it will only improve tracking for visitors who've already consented; it will not close this gap for visitors who decline.
  • The gap above also isn't purely "sales lost to consent": it includes traffic from completely unrelated non-Meta sources too, which WooCommerce's attribution can't separate out.
  • Treat the WooCommerce-attributed figures as the more reliable performance measure going forward.

Actions This Week

Meta Ads changes made Jul 1–7, 2026.

✎

Attribution Gap Scope Narrowed With Graphedia

Graphedia confirmed consent-declining visitors can't be legally tracked back to an ad by any method, including the proposed server-side Conversions API fix, under Irish ePrivacy/GDPR rules. Agreed to proceed with a CAPI build that respects existing consent rather than push for a workaround. Now understood as a permanent limitation to report around, not a bug to keep chasing.

✓

BOF Retargeting Frequency Resolved

Frequency dropped to 4.74 from 5.02 following the Jun 29 audience expansion to 90 days: back under the 5.0 healthy threshold. No further action needed this week.
